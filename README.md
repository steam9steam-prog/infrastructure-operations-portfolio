# Портфолио по инфраструктуре и эксплуатации

Портфолио по VPS-инфраструктуре и эксплуатации сервисов: деплой, nginx, Docker, PostgreSQL, systemd, мониторинг, логи, бэкапы и разбор инцидентов.

Здесь нет приватного кода, токенов, реальных конфигов, IP-адресов, доменов и дампов базы. Репозиторий нужен, чтобы показать мой подход к инфраструктуре живых сервисов и рабочим задачам по серверам.

## Кто я

Middle Infrastructure / Operations Engineer

Настраиваю и сопровождаю VPS-инфраструктуру для Telegram-ботов, mini apps и веб-сервисов.

## Практический кейс

В качестве примера внутри разобрана инфраструктура моего живого VPN-сервиса. Это не основное направление продаж, а удобный реальный кейс, где есть несколько типичных компонентов:

- сайт
- Telegram-бот
- Telegram mini app
- админка
- backend/control-сервис
- PostgreSQL
- nginx + SSL
- systemd/Docker
- Grafana/Prometheus
- логи, проверки здоровья сервиса, алерты

## Что я реально делаю руками

- поднимаю сервисы на VPS
- настраиваю nginx и SSL
- запускаю сервисы через Docker Compose или systemd
- работаю с .env и переменными окружения
- смотрю journalctl, nginx logs и docker logs
- проверяю падения после деплоя
- делаю backup/restore для PostgreSQL
- добавляю проверки здоровья сервиса и рабочие метрики
- разбираю инциденты: что упало, почему, как починить

## Зачем этот репозиторий

Это не рабочий репозиторий сервиса и не исходный код приложения.

Это портфолио, которое показывает:

- как устроена инфраструктура живого сервиса
- какие компоненты нужны для деплоя Telegram/web/VPS-проекта
- как я проверяю сервис после изменений
- какие команды использую для диагностики
- как отношусь к секретам и безопасности
- как оформляю понятный runbook на случай падения

## Что можно проверить

- есть схема архитектуры и маршрутизации
- есть порядок деплоя через Docker Compose и systemd
- есть nginx example для API/webhook и HTTPS
- есть пример systemd service
- есть пример Docker Compose
- есть список метрик и алертов
- есть инструкция для разбора падений

## Примеры внутри

В examples/ лежат не реальные конфиги, а безопасные шаблоны:

- nginx reverse proxy
- systemd service
- Docker Compose
- Prometheus alert rules

Везде используются заглушки:

- example.com
- 127.0.0.1
- CHANGE_ME
- тестовые имена сервисов
- тестовые порты

## Какие задачи закрываю

- настроить VPS под проект
- задеплоить Telegram-бота или mini app
- настроить nginx + SSL
- завернуть сервис в Docker Compose
- сделать systemd service
- настроить PostgreSQL backup
- добавить Grafana/Prometheus
- посмотреть логи и понять, почему сервис падает
- подготовить инструкцию на случай падения
- сопровождать проект на постоянке

## Безопасность

В публичный репозиторий нельзя выкладывать:

- Telegram bot token
- payment/webhook secrets
- реальные .env
- приватные nginx configs
- IP-адреса серверов
- пароли от PostgreSQL
- дампы базы
- боевые ключи

В этом портфолио все такие данные заменены на безопасные примеры.
